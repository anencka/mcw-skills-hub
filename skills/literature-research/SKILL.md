# Literature & Background Research

## When to Use
The operator needs to understand a topic, find prior work, build a background/related-work
section, or fact-check claims with traceable sources.

## First-use setup (read the profile; fill if thin)
Read `~/work-profile/domain-knowledge.md` (field vocabulary, key authors/venues, what not to
over-explain) and `~/work-profile/current-projects.md` (what the operator is working on, to focus
the search). If either is still mostly `*(to capture)*`, offer **once** to fill it — a few
questions (hand to `onboarding-interview`, e.g. "let's do domain-knowledge") or the operator edits
it in the Files tab — then proceed; if they skip, note that in memory and don't re-ask. **Read
these fresh each use; never cache their contents — the file is the source of truth.** Preferred
databases + citation style/manager live in `tools-and-systems.md` (hand off to
`reference-management`); ask in-task if absent.

## Procedure
1. **Clarify the question** into specific, answerable sub-questions; state scope and what
   "done" looks like.
2. **Search broadly, then narrow** — use the operator's preferred sources; capture each result's
   title, authors, year, venue, and URL/DOI as you go (never reconstruct citations from memory).
3. **Read, don't skim-quote.** For each kept source, extract the specific claim/evidence and the
   location. Distinguish primary findings from secondary commentary.
4. **Synthesize by theme**, not by source. Every nontrivial claim gets a citation. Surface
   disagreements and gaps, not just consensus.
5. **Flag uncertainty** explicitly: what's well-supported, what's thin, what you couldn't verify.
6. **Deliver** a structured brief (question → findings by theme → gaps → full source list with
   DOIs/URLs). Offer to hand the sources to `reference-management`.

## Pitfalls
- Fabricated or "plausible" citations — the cardinal sin. If you didn't open it, don't cite it.
- Over-trusting a single review; triangulate primary sources.
- Synthesizing source-by-source (a literature dump) instead of by theme.
- Letting a cheap model summarize without reading the source text.

## Verification
Every claim in the brief maps to a real, openable source in the source list; uncertainties are
labeled; the operator can trace any statement back to its origin.
