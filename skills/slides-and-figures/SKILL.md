# Presentations & Figures

## When to Use
Outlining or building a talk (conference, lab meeting, seminar, defense), preparing slide figures,
or adapting a paper/figure for presentation.

## First-use setup (read the profile; fill if thin)
Read `~/work-profile/communication-style.md` (voice, formatting),
`~/work-profile/team-and-relationships.md` (typical audiences), and
`~/work-profile/domain-knowledge.md` (how much to explain). If any is still mostly
`*(to capture)*`, offer **once** to fill it — a few questions (hand to `onboarding-interview`,
e.g. "let's do communication-style") or the operator edits it in the Files tab — then proceed; if
they skip, note that in memory and don't re-ask. **Read these fresh each use; never cache their
contents — the file is the source of truth.** Tooling/template and talk length are talk-specific —
ask for the task at hand.

## Procedure
1. **Story first, slides second.** Establish the through-line: problem → why it matters → what you
   did → key result → takeaway. Outline before building.
2. **One idea per slide.** A clear assertion headline (the point, not a topic), minimal text,
   supported by one figure/visual. Cut anything that doesn't serve the message.
3. **Slide figures ≠ paper figures.** Bigger fonts, fewer panels, high contrast, legible from a
   distance; reduce/redraw dense paper figures for the room.
4. **Respect time:** ~1 slide/minute as a rough budget; flag when content exceeds the slot.
5. **Build in the operator's tool**, applying their template; keep a consistent visual system
   (fonts, colors, spacing).
6. **Speaker support:** offer concise speaker notes / talking points per slide if wanted.

## Pitfalls
- Wall-of-text slides; topic headlines instead of assertion headlines.
- Pasting paper figures unchanged (tiny fonts, too many panels).
- No narrative — a pile of results with no arc or takeaway.
- Overrunning the time slot.

## Verification
The deck has a clear narrative arc and a single point per slide, figures are legible at
presentation scale, it fits the time budget, and it uses the operator's tool/template.
